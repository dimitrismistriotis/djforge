### Rationale

After much deliberation, we decided to use a licensing model rather than go fully open source. The reason is simple: open source contributions often go unappreciated, particularly for a project with direct commercial value. Without some friction, most people and companies would happily fork or clone the repo with no intention of contributing back — or worse, return later demanding bug fixes.

At the same time, we are aware that in the current AI-assisted development era, large portions of a project like this could theoretically be generated by an LLM. With both of those realities in mind, we landed on a symbolic fee — "pay, but not much." We believe it's faster to start from this project and strip out what you don't need than to vibe-code your way to the same result. And because the codebase follows clean Django conventions, it'll naturally guide any LLM you use toward writing better code too.

Thanks for reading, and for your support.

### Editors

We use the community edition of [PyCharm](https://www.jetbrains.com/pycharm/) and [VS Code](https://code.visualstudio.com/).

Settings are included and stored in the `.idea` and `.vscode` folders respectively. You might want to change them
or completely remove them from the project based on which IDE/editor you use and how.
After some deliberation, we decided to include them in order to make development faster
for those who use or will use either of these editors.

## Postgres

We decided to use Postgres as the database for this project as it is the most popular
RDBMS for Django projects, also one of the most popular RDBMS in general.

For most applications a RDBMS is a good-enough choice, which makes it a good default
to kickstart your project.
